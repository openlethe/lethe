# Support

## Questions and bug reports

Use a GitHub issue for reproducible bugs, documentation gaps, and narrowly scoped feature requests. Include:

- Lethe version, image tag, or commit;
- operating system or container runtime;
- deployment mode (`legacy` or `git`) and trust mode, without secret values;
- relevant sanitized logs;
- expected and observed behavior;
- a minimal reproduction when possible.

Do not include API keys, HMAC keys, tokens, private memory content, database files, or personal data.

## Security issues

Follow [SECURITY.md](SECURITY.md). Do not publish exploit details in an issue or discussion.

## Scope

Community support covers the current release and `main`. Deployment-specific reverse proxies, container platforms, and private modifications may require reproduction against the repository's documented reference configuration.

Lethe is provided without a commercial support guarantee. Maintainers may prioritize security, correctness, and reproducible defects over environment-specific integration requests.
